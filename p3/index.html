WHAT IS RUST?

Rust is a programming language focused on safety, control of memory layout, and concurrency.

It is syntactically similar to C and C++, but designed to provide better memory safety while maintaining high performance.

Safety & Memory Layout

Rust's compiler allocates memory and manages it so objects are manageed from beginning to end.

Concurrency
Definition: The ability of different parts of a program, algorithm, or problem to be executed out-of-order or in partial order, without affecting the final outcome.

Rust is able to run multiple concurrent units in a program to improve overall speed of processes.

A BRIEF HISTORY

Rust first started as a personal project begun in 2006 by a Mozilla employee. It was eventually sponsored by Mozilla in 2009.

The first stable release of Rust 1.0 was in 2015. New stable point releases are launched every six weeks usually with significant changes with each version.

People enjoy coding in Rust and it was even ranked the third-most-loved programming language in the 2015 Stack Overflow annual survey and received first place in 2016–2019.

WHY RUST?

SAFETY! MEMORY! CONCURRENCY!

  1. The developer doesn’t have to manage memory like in C or C++ programs.
    • The proper amount of memory is allocated so when its job is done, the object will automatically be deallocated by the system. This means a programmer does not have to manually unreserve the memory used by that object.
    • It’s almost impossible to accidentally access a memory location that is out of bounds.
  2. Unmanaged code is often harder to write and more vulnerable to threats.
    • Buffer overflows, for example, are a type of threat where a user can enter more information than can be contained within the program’s allotted memory space, potentially allowing for the modification of memory in parts of the system not under direct control of the code. Rust’s capabilities prevent these type of threats.
  3. Rust’s compiler checks every variable you use and every memory address you reference.
    • In dangerous programs, you’d be fighting the compiler, since almost every operation you try would cause memory safety issues.
    • The compiler solves all issues at compile time eliminating the need for a garbage collector that would slow down a program.
Pros


- Sloppy code is difficult to write because of Rust's check on memory allocation which doubles as a safety check for potential threats.

- If you manage to appease the compiler, your code will likely work.

Cons


- The compile messages are helpful but can be tedious.

- Rust is still a new language so some of its libraries are incomplete.

INSTALLING RUST

You can Rust through rustup, a command line tool for managing Rust versions and associated tools.

In Terminal (For Linux and Mac): 

        $ curl https://sh.rustup.rs -sSf | sh
          Rust is installed now. Great!

CREATING A PROJECT WITH CARGO

Cargo is Rust’s build system and package manager.

Cargo is used to manage Rust projects because it handles a lot of tasks, such as building code, downloading the libraries the code needs (dependencies), and building those libraries. 

        $ cargo new hello_cargo
        $ cd hello_cargo

The first command creates a new directory called hello_cargo. The project is named hello_cargo, and Cargo creates its files in a directory of the same name.

Cargo will automatically generated two files and one directory: a Cargo.toml file and a src directory with a main.rs file inside.

Filename: Cargo.toml (Tom’s Obvious, Minimal Language) 

        [package]
        name = "hello_cargo"
        version = "0.1.0"
        authors = ["Your Name "]
        edition = "2018"

        [dependencies]

Filename: src/main.rs 

        fn main() {
          println!("Hello, world!");
        }

Cargo expects source files to live inside the src directory. The top-level project directory is just for README files, license information, configuration files, and anything else not related to your code. Using Cargo helps you organize your projects.

BUILDING AND RUNNING A CARGO PROJECT

The cargo build command creates an executable file in target/debug/hello_cargo rather than in your current directory. 

        $ cargo build
          Compiling hello_cargo v0.1.0 (file:///projects/hello_cargo)
            Finished dev [unoptimized + debuginfo] target(s) in 2.85 secs

You can run the executable with: 

        $ ./target/debug/hello_cargo #
        Hello, world!

You can also use cargo run to compile the code and then run the resulting executable all in one command: 

        $ cargo run
          Compiling hello_cargo v0.1.0 (file:///projects/hello_cargo)
            Finished dev [unoptimized + debuginfo] target(s) in 0.33 secs
              Running `target/debug/hello_cargo`
          Hello, world!

Cargo also provides a command called cargo check. This command quickly checks your code to make sure it compiles but doesn’t produce an executable: 

        $ cargo check
          Checking hello_cargo v0.1.0 (file:///projects/hello_cargo)
            Finished dev [unoptimized + debuginfo] target(s) in 0.32 secs

Often, cargo check is much faster than cargo build, because it skips the step of producing an executable. If you’re continually checking your work while writing the code, using cargo check will speed up the process.
